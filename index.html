MendelBase.jl

MendelBase contains the base functions of OpenMendel.

Download this project as a .zip file Download this project as a tar.gz file

Overview

MendelBase is a utility package that contains the base functions of OpenMendel. It provides the information needed to process keywords and read data from external files. MendelBase is necessary to run any analysis package, and you should install MendelBase first, before you install the other packages.

Installation

Within Julia, use the package manager to install MendelBase:

Pkg.clone("git@github.com:OpenMendel/MendelBase.jl.git")

This package supports Julia v0.4.

Input Files

To make your genetic data available to OpenMendel, the various input files must be in a format that OpenMendel can read. To provide OpenMendel with the information it needs to perform its analysis you must prepare the following files. Examples input files can be found in the docs subfolders of the analysis packages. (An analysis option won't always need every file type.)

  • Control File: Specifies the names of your data input and output files and any optional parameters (keywords) for the analysis. (For a list of common keywords, see Keywords Table below).
  • Locus File: Names and describes the genetic loci in your data.
  • Pedigree File: Gives information about your individuals, such as name, sex, family structure, and ancestry. This may be a PLINK FAM format file.
  • Phenotype File: Lists the available phenotypes.
  • SNP Definition File: Defines your SNPs with information such as SNP name, chromosome, position, allele names. This may be a PLINK BIM format file.
  • SNP Data File: Holds the genotypes for your data set. This must be a standard binary PLINK BED file in SNP major format. If you have a SNP data file you must have a SNP definition file.

Control files

The Control file is a text file consisting of keywords and their assigned values. The format of the Control file is:

Keyword = Keyword_Value(s)

Below is an example Control file to run Gamete Competition (text after "#" is ignored, and treated as comments):

#
# Input files.
#
locus_file = gamete competition LocusFrame.txt
pedigree_file = gamete competition PedigreeFrame.txt
#
# Analysis parameters.
#
disease_status = ACE
affected_designator = 1
standard_errors = true

In the example above, there are five keywords. The first two specify input and output files: "gamete competition LocusFrame.txt", "gamete competition PedigreeFrame.txt". The last three specify analysis parameters: disease_status, affected_designator, standard_errors. The text after the '=' are the keyword values: ACE, 1, true. For keywords that can take a set of values, the values are separated by commas.

General Data File Format

All data files are in text file format with one exception: if you have dense SNP data, the SNP data should be in binary files in PLINK BED file format. OpenMendel will also accept PLINK format FAM and BIM files. All non-PLINK OpenMendel data files list data in columns, with a header row with field names for each column. The columns need not be in any fixed order. Many column headers have required, case sensitive names. The column separators can be any single character, specified in the control file with the keyword field_separator. The default separator is comma. If a data element contains the field separator (e.g. comma) it must be enclosed in double quotes, or it will be read as a new field. Missing data is blank ("") or "NA". The PLINK file missing data symbols are "0" or "-9".

Pedigree File

A Pedigree file is always required. OpenMendel will accept PLINK format FAM Pedigree files, in which case the file must have the extension .fam. The OpenMendel Pedigree file format has a header row with the following field names (in any order): Pedigree,Person,Mother,Father,Sex,Twin. The only required column is one labeled either “Individual” or “Person”, giving the samples names. Each individual must either have both and mother and a father, or both mother and father must be blank. If the Mother and Father columns exist but both fields are missing for an individual, that individual is treated as a founder. If the Mother and Father columns are missing, all individuals are founders. If the Pedigree column is missing, the data is treated as separate pedigrees of size one, and all individuals are unrelated. Within each pedigree, every person must have a unique name. If a set of individuals in the same pedigree all have the same (non-blank) value in the Twin field they are all considered monozygotic (MZ) siblings. If the Sex field is missing, all individuals are treated as female. If a Sex column exists but the sex is missing for an individual, that individual is treated as female. If the analysis uses a Locus file to describe the markers, the genotypes are contained in the Pedigree file, and the marker names in the Locus and Pedigree files must agree. If the analysis uses a SNP Definition file to describe the markers, the genotypes are contained in the SNP Data file. The Pedigree file may also include optional columns such as traits and population ancestries.

Pedigree,Person,Mother,Father,Sex,Twin,ABO,HtMeters
Clinton,Bill,,,male,0,A,1.88

Obama,Michelle,,,female,0,B,NA
Obama,Malia,Michelle,Barack,female,0,O,NA

Locus File

The Locus file defines your genetic markers if you are working with a relatively small number of markers. If you have a dense SNP map you should use a SNP Definition file to define your SNPs. The Locus file must have a header row with the field names (in any order): Locus, Allele, Chromosome, Frequency. You may also include locus position information with optional the fields Morgans, FemaleMorgans and MaleMorgans, and Basepairs. For each locus there will be one line per locus allele.

Example Locus File

Locus,Allele,Chromosome,Frequency
ABO,A,9,0.64,0.27
ABO,B,9,0.34,0.06
ABO,O,9,1.35,NA
Rh,D,1,2.42,0.61
Rh,d,1,0.58,0.39
SNP,1,9,0,NA
SNP,2,9,0,NA

SNP Definition File

The SNP Definition file defines your SNPs with information such as SNP name, chromosome, position, and allele names. OpenMendel will accept PLINK format BIM files, in which case the file must have the extension .bim. The OpenMendel SNP Definition file format has a header row with the field names: SNP or Locus (if this column is absent the SNPs are named 1,2,3..n); Chromosome (1-22,x; if column is absent assumed autosomal); Locus, CentiMorgans (if column is absent uses 0); Basepairs (if column is absent uses 0); Allele1, Allele2 = allele names. If allele names are absent OpenMendel uses "1" for Allele1 and "2" for Allele2.

Example SNP Definition File

Chromosome,Locus,CentiMorgans,Basepairs,Allele1,Allele2
X,rs311165,0,2699555,C,A
X,rs28579419,0,2699645,T,G
X,rs2306736,0,2700027,C,T
X,rs5939320,0,2700202,A,G
X,rs4892819,0,2700613,G,A
X,rs5982853,0,2702143,C,T
X,rs73433431,0,2702698,T,C

SNP Data File

The SNP Data file holds the SNP genotypes for your data set. The SNP Data file must be a standard PLINK format binary BED file in SNP major format. If you have a SNP data file you must have a SNP definition file, but it need not be in PLINK format. (There is no example SNP Data file because the SNP Data file is a binary file, and not human readable.)

PLINK Basename

You may use the keyword plink_input_basename in the Control file to specify a name for your PLINK format files. If you use this keyword you do not need to name each PLINK file in the Control file, but you must supply each of the three relevant PLINK files, using that basename: your_basename.bed (SNP data file), your_basename.bim (SNP definition file), and your_basename.fam (pedigree file).

Populations

If your individuals are of mixed ethnicity, and thus have fractional ancestry, OpenMendel will allow you to adjust for this in your analysis by using the populations keyword. Your populations will be named in your control file, fractional ancestries assigned to your individuals in the Pedigree file, and population specific allele frequencies assigned to markers in your Locus file.

For example, your Control file might look like this:

# Input and Output files.
#
locus_file = genedropping LocusFrame.txt
pedigree_file = genedropping PedigreeFrame.txt
phenotype_file = genedropping PhenotypeFrame.txt
new_pedigree_file = genedropping NewPedigreeFrame.txt
#
# Analysis parameters for GeneDropping option.
#
repetitions = 2
populations = European, African, Chinese

Your Pedigree file would have a column for each population, with a fractional ancestry assigned. The fractional ancestries need not sum to 1, but will be normalized to 1 in the analysis.

An example Pedigree file might look like this:

Pedigree,Person,Mother,Father,Sex,Twin,European,African,Chinese,ABO
Clinton,Bill,,,male,0,1,0,0,A
Clinton,Hillary,,,female,0,1,0,0,O
Clinton,Chelsea,Hillary,Bill,female,0,NA,NA,NA,A
Obama,Barack,,,male,0,0.5,0.5,0,O
Obama,Michelle,,,female,0,0.3,0.7,0,B
Obama,Malia,Michelle,Barack,female,0,NA,NA,NA,O
Obama,Sasha,Michelle,Barack,female,0,NA,NA,NA,B

Your Locus file would have, instead of one column labeled Frequency, a frequency column for each population, where the header for the column is the name of that population, and loci have an allele frequency for each population. The population frequency field names are the population names specified in the control file. In the example Locus file below, the Control file would include the line "populations = European,African,Chinese". The population frequencies need not sum to 1, but will be normalized to 1 in the analysis.

An example Locus file might look like this:

Locus,Allele,Chromosome,European,African,Chinese
ABO,A,9,0.27,0.18,0.19
ABO,B,9,0.06,0.11,0.17
ABO,O,9,NA,0.71,0.64

Output Files

All OpenMendel analysis options will create a results file. If no name is for this file is specified in the Control, it will be given the default name Mendel_Output.txt. A number of OpenMendel options give you the option of creating output files, such as plot files, or new pedigree, locus, or SNP input files based on the analysis results. These newly created input files can be used in subsequent analysis runs. To create these files, the file type is specified by a keyword in the control file, and a filename may be specified.

For example:

#
# Input and Output files.
#
plink_input_basename = gwas 1 data
output_file = gwas 1 Output.txt
manhattan_plot_file = ManPlotOutput_1.png
#
# Analysis parameters for GWAS option.
#
regression = linear
regression_formula = Trait ~ Sex
