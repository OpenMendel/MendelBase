Input Files

To make your genetic data available to OpenMendel, the various input files must be in a format that OpenMendel can read. To provide OpenMendel with the information it needs to perform its analysis you must prepare the following files. (An analysis option won't always need every file type.)

  • Control File: Specifies the names of your data input and output files and any optional parameters (keywords) for the analysis. (For a complete list of keywords, see Keywords Table below).
  • Locus File: Names and describes the genetic loci in your data.
  • Pedigree File: Gives information about your individuals, such as name, sex, family structure, and ancestry.
  • Phenotype File: Lists the available phenotypes.
  • SNP Definition File: Defines your SNPs with information such as SNP name, chromosome, position, allele names, allele frequencies. This may be a PLINK .bim format file.
  • SNP Data File: Holds the genotypes for your data set. Must be a standard binary PLINK .bed file in SNP major format. If you have a SNP data file you must have a SNP definition file.

Control files

The format of the Control file is

Keyword = Keyword_Value(s)

Below is an example Control file to run GWAS (text after "#" is commented out):

#
# Input and Output files.
#
plink_input_basename = gwas 1 data
output_file = gwas 1 Output.txt
manhattan_plot_file = ManPlotOutput_1.png
#
# Analysis parameters.
#
regression = linear
regression_formula = Trait ~ Sex

In the example above, there are five keywords. The first three specify input and output files: plink_input_basename, output_file, manhattan_plot_file. The last two specify analysis parameters: regression and regression_formula. The text after the '=' are the keyword values.

Locus File

The Locus File must have a header row with the fieldnames (in any order): Locus, Allele, Chromosome, Frequency. If your samples come from different populations, instead of one column lablelled Frequency, name the populations in the control file with the keyword populations, and give a frequency for each population, where the fieldnames are the population names specified in the control file.

Example

Locus,Allele,Chromosome,Basepairs,FemaleMorgans,MaleMorgans,European,African,Chinese
ABO,A,9,10,0.0,0.0,0.27,0.18,0.19
ABO,B,9,10,0.0,0.0,0.06,0.11,0.17
ABO,O,9,10,0.0,0.0,NA,0.71,0.64
Rh,D,1,0,0.0,0.0,0.61,0.81,1.0
Rh,d,1,0,0.0,0.0,0.39,0.19,0.0
SNP,1,9,0,0.2,0.1,NA,NA,NA
SNP,2,9,0,0.2,0.1,NA,NA,NA

Pedigree File

A Pedigree file is always required. It can be in PLINK FAM file format, in which case it must have the extension .fam, or it can have the format of a required header rows with the following fieldnames (in any order): Pedigree,Person,Mother,Father,Sex,Twin, and optional columns such as populations ancestries.

Pedigree,Person,Mother,Father,Sex,Twin,BirthYear,Politician,European,African,Chinese,ABO,Rh,Xg,XSNP,HtMeters

SNP Definition File

Defines your SNPs with information such as SNP name, chromosome, position, allele names, allele frequencies. This may be a PLINK .bim format file.

SNP Data File

Holds the genotypes for your data set. Must be a standard binary PLINK .bed file in SNP major format. If you have a SNP data file you must have a SNP definition file.

The symbol for missing data is "" or "NA". The PLINK File missing data symbols are "0" or "-9"
